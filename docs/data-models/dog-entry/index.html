Ticket to the Tourney

This is the actual thing that says "this dog runs in this trial."

PropertyData TypeNotes
Entry IDidPrimary key for the table.
Dog's IDidForeign key from Dog table.
Trial IDidForeign key from Trial table.
Owner's IDidForeign key from Users table.
HandleridUsually the same as owner but can be someone else. (If this person is not a user, I can't really use an id here.)
Junior Handlerboolean
Regular or PreferredenumRegular, Preferred. To clarify this distinction, if you're running your dog in a preferred class, your jumps are one level LOWER than what they would be at standard. Usually older dogs do this.
Days Participatingarray of trialDayPick the days available from Trial with same Trial id
Classes EnteredenumStandard, Jumpers With Weaves, Time 2 Beat, FAST picking what's available from this Trial
Level for Classes EnteredenumSee Ring model to get an idea of which Levels are available for each Class
Jump HeightstringReferenced from Dog table by default, but be able to change it here, leaving the original alone. (Prefill with the data when signing up)
Running OrderstringWhere in the order is the dog running in each Class she entered?
ArmbandstringAssigned automatically, concatenating Jump Height with running order.
Move Up?BooleanA move up is when a dog reaches a new level at a different trial, prior to this trial happening. Now, the dog that signed up for one level, needs to be moved up to a new one. If the trial is still open for entries, the user can switch the level on their own. But if not, they need to send a note to the Secretary so that she can make the changes. When this happens, we need to keep track of it with a boolean.
