It's All Made Up and the Points Don't Matter

During a trial, there's going to be a list of dogs in their Running order. One by one, they will run and a trial admin will input the dog's score.

PropertyData TypeNotes
Dog IDidThe id of the dog running this particular run.
Run IDidThis is the id of THIS particular run. The same dog can have several different runs a day, in different classes. This is just for this run
Scorekeeper IDidWe need to know who kept score of each run. This is going to be a trial sub-secretary who has their own login and profile
ClassstringWhich class is this run taking place in?
LevelstringWhich level is this run taking place in?
HeightstringWhich height is this run taking place in?
Armband NumberstringThis is an archaic uuid, but for some stupid reason, it's required by AKC. It's a 5 digit number, with the first two digits representing the dog's jump height, and the last three digits repping the dog for the whole day. SO, 12069 would be a 12 inch dog, running 69th of the day. This can be calculated once trial secretary locks in the order for the trial in the days before trail day
Trial nametrialDay
TimestampdateA timestamp from when the score was submitted
